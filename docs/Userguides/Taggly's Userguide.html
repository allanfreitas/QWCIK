Taggly

Taggly is a customizable CI library that makes creating tag clouds a snap.

Setting-Up Taggly

Place the Taggly.php library file in the following directory (default libraries directory):

www-root/system/libraries/

Then, load up the Taggly library as you would with any other CodeIgniter library. There are two possible ways to do this. Directly from within your controller:

$this->load->library('taggly');

Or, auto-load the Taggly library from within system/application/config/autoload.php:

$autoload['libraries'] = array('database', 'session', 'taggly');

Using Taggly

cloud($tagsArray, $configArray="")

Generates and returns HTML for a tag-cloud based on data given.

$tagsArray:
This is a two-dimensional array that contains the data for each tag. The required information is Tag Count, Tag Name and Tag URL respectively. $tagsArray = array(array($count, $name, $url))

$configArray:
This is an optional field. It allows you to set configurations within Taggly using an array. See Configuring Taggly for available options.

Basic Usage: $tagsArray = array (
    array(10, 'Dog', 'http://dogs.com'),
    array(53, 'Cat', 'http://cats.com')
);

echo $this->taggly->cloud($tagsArray);

Advanced Usage (passing configurations): $configArray = array (
    'min_font' => 20,
    'max_font' => 60,
    'html_start' => '<span>',
    'html_end' => '</span>',
    'shuffle' => FALSE,
    'class' => 'my_tag_class',
    'find_Match' => $matchTags,
    'match_Class' => 'bold',
);

$tagsArray = array (
    array(10, 'Dog', 'http://dogs.com'),
    array(53, 'Cat', 'http://cats.com')
);

echo $this->taggly->cloud($tagsArray, $configArray);


Configuring Taggly

config($configArray)

$configArray:
An array containing available Taggly options. The available options are:

  • min_font - Set minimum tag font size. Must be an integer. Defaults to 12
  • max_font - Set maximum tag font size. Must be an integer. Defaults to 24
  • html_start - Put HTML at beggining of each tag. Must be a string.
  • html_end - Put HTML at end of each tag. Must be a string.
  • shuffle - Randomly shuffle the tag order. Must be a boolean value. Defaults to TRUE
  • class - A CSS class name to give each tag. Must be a string. Defaults to 'taggly'
  • find_Match - Set array of matching Tags. [0]=>blaat etc..
  • match_Class - A CSS class name for each matching tag.

With "find_Match" you can find matching tags within your cloud. This enables you to highlight corresponding tags in the cloud. The "match_Class" option sets the class for a matching tag. The "find_Match" contains the names of the normal Taggly array: array(10, 'Dog', 'http://dogs.com'),
 The "find_Match" looks like: $matchTagsArray =array(Cats, 'Dog', 'Birds');

Example:

$matchTagsArray =array(Cats, 'Dog', 'Birds');

$configArray = array (
    'min_font' => 20,
    'max_font' => 60,
    'html_start' => '<span>',
    'html_end' => '</span>',
    'shuffle' => FALSE,
    'class' => 'my_tag_class',
    'find_Match' => $matchTagsArray,
    'match_Class' =>"boldTag",
);

$this->taggly->config($configArray);

$tags1 = array (
    array(10, 'Dog', 'http://dogs.com'),
    array(53, 'Cat', 'http://cats.com')
);

$tags2 = array (
    array(5, 'Fish', 'http://fish.com'),
    array(27, 'Horse', 'http://horse.com')
);

echo $this->taggly->cloud($tags1);
echo $this->taggly->cloud($tags2);
